## AUTHENTICATION

### Signup (Mobile & Extension)
- Endpoint: POST /auth/signup
- Required: email, name, password, passwordConfirm
- Optional: age
- Response: { user, token }
- Store JWT in secure storage
- Mobile: After signup, show QR code screen "Scan to connect extension"
- Extension: After signup, redirect to home

### Login
- Email login: POST /auth/login (email, password)
- QR login (extension only):
  - Extension shows QR scanner
  - User scans QR generated from mobile app (5-min expiry)
  - Exchange code: POST /auth/qr-login → return JWT

### QR Connection (Mobile → Extension)
- Mobile generates 5-min expiry code
- Extension scans, sends to backend
- Backend links devices, returns shared session state
- Sync all active/upcoming sessions after login

### Extension First Load (Not Logged In)
- Show two options: "Login with Email", "Login with QR Code"
- Show link: "New user? Get the app!" → QR code linking to app download

## FOCUS SESSION MANAGEMENT

### Create Session (Mobile Only)
- Fields:
  - name (string, required)
  - duration (minutes, required)
  - schedule: "today" | "specific_days" | "recurring"
  - scheduleDays (array of weekdays, if specific_days or recurring)
  - startTime (time, required)
  - mobileFocus (boolean toggle)
  - browserFocus (boolean toggle)
- If mobileFocus = true: show app group selector/creator
  - Select existing group OR create new group
  - Enter group name, select distracting apps, save
- POST /sessions
- If browserFocus OFF and extension not connected: show notification "Session started without browser focus"
- On success: show session on home screen

### Edit Session (Mobile Only)
- Edit any field (name, duration, schedule, toggles, app groups)
- PUT /sessions/:id
- Sync update to extension in real-time

### Delete Session (Mobile Only)
- DELETE /sessions/:id
- Remove from mobile and extension UI

### Cancel Active Session (Mobile & Extension)
- Click red X button on session card
- Show confirmation modal: "Cancel this session?" → [Cancel] [Confirm]
- If confirmed:
  - POST /sessions/:id/cancel
  - Log: timestamp, time remaining, violations count, optional cancellation reason
  - Stop all monitoring
  - Move to history

### Session Display (Mobile Home)
- Group: Active (top) → Upcoming (middle) → Scheduled (rest)
- Each card shows: name, time/duration, mobile/browser toggle icons, edit/delete
- Click card: view details, edit (if not active), cancel (if active)

### Session Display (Extension Popup)
- Active session card (larger):
  - Name, countdown timer (hh:mm:ss), violations count, red X cancel button, ☰ menu
- Below: Upcoming session card (smaller): name + start time only
- Real-time sync: poll every 5 seconds

### Session Overlap Prevention
- Prevent creating sessions with overlapping times
- Show validation error if overlap detected

### Offline Handling
- If extension disconnects during session: show "Browser focus inactive" in mobile app
- Queue violations if offline, sync when back online

## VIOLATION DETECTION & LOGGING

### Mobile (Blocked Apps)
- Continuously monitor foreground app
- Check against active session's app groups (if mobileFocus ON)
- On blocked app detected:
  - Start 5-second timer silently
  - If user closes/switches app before 5sec: cancel timer, no violation logged
  - If still open after 5sec:
    - Show full-screen translucent red overlay (rgba(220, 53, 69, 0.6))
    - White warning triangle (▲) centered
    - Play 2-3 second beep sound + 200ms vibration (first detection only)
    - Show 3 options:
      - "Allow anyway 🫥" → violation logged, session continues, track duration
      - "Mark as necessary" → add to session allowlist, no violation logged, won't trigger again this session
      - (User closes app → no violation logged)

### Extension (Blocked Tabs)
- Monitor tab activity via background.ts (tabs.onActivated, tabs.onUpdated)
- Check current URL against blocked websites list
- Same 5-second timer logic as mobile
- Same red overlay + prompt injected via content.ts
- Same 3 options

### Violation Data Logged
- sessionId
- appName or websiteName
- timestamp (when 5sec expired)
- durationSeconds (how long on blocked app/site)
- actionTaken: "allow_anyway" | "mark_necessary" | "closed"
